# 导线平差计算 App

> 🚀 **在线体验网址**：[https://yiyil8020-ux.github.io/daoxian-pingcha/](https://yiyil8020-ux.github.io/daoxian-pingcha/)

闭合 / 附合导线测量内业计算的移动端 Web App。纯 HTML/CSS/JS，零依赖，可部署到 Vercel / Netlify / GitHub Pages / 任意静态服务器。


## 功能

- ✅ 闭合 / 附合 双模式（顶栏下方 sticky 切换，永远可见）
- ✅ 3 栏度分秒角度输入
- ✅ 左角 / 右角 切换
- ✅ 起算方位角 3 种输入：度分秒 / 十进制 / 两点反算（atan2）
- ✅ 终止方位角同理（附合模式）
- ✅ **按「🚀 计算」按钮触发平差**（不实时重算；改输入后按钮变橙脉动，提示「已修改 · 点此重算」）
- ✅ **测站角度表 + 边长表 拆成两张独立表**（边长是线段，不再挂在点号上；标签「A → B」只读，随点号自动更新）
- ✅ 角度闭合差 + 全长相对闭合差限差校验
- ✅ **超限不阻断**：仅标红 + 警告条，继续给出全部平差结果
- ✅ Excel 风格结果表（14 列，仿 `导线平差计算表.xlsx`）
- ✅ **整数修正模式**（v_β 整秒、vx/vy 1mm 整数倍；余量按 |β| / 边长 largest-remainder 分）
- ✅ **v_β 列**独立显示角度改正数（带符号，整数模式整秒 / 小数模式 1 位）
- ✅ **X' / Y' 未修正坐标增量**独立显示（带撇 = 调整前；跟 vx/vy 并列，便于人工核算）
- ✅ Canvas 略图（自动适配 + 比例尺）
- ✅ localStorage 自动保存草稿
- ✅ 方案管理（命名保存 / 列表载入 / 删除）
- ✅ 导出：TSV（粘到 Excel，14 列对齐）/ PNG（结果表截图）/ JSON（完整备份）
- ✅ PWA：可"添加到主屏幕"当 App 用
- ✅ 离线可用（Service Worker 缓存）

## 坐标约定

- **X = 纵坐标 = 北**
- **Y = 横坐标 = 东**
- 方位角 α 从北起顺时针

> 此为测量学常用约定，与 Excel 表中 X/Y 列对应。

## 快速开始

### 本地运行

```bash
cd traverse-calc
python3 -m http.server 8765
# 浏览器打开 http://localhost:8765
```

或直接双击 `index.html` 打开（部分浏览器对 ES Module + SW 有限制，推荐 HTTP server）。

### 部署到 Vercel

**方式 A：CLI（推荐）**

```bash
cd traverse-calc
npx vercel@latest login        # 首次需登录（会跳转浏览器）
npx vercel@latest              # 部署到预览环境
npx vercel@latest --prod       # 部署到生产环境
```

部署完成会得到一个 `https://xxx.vercel.app` 链接，手机/电脑都能直接打开。**首次部署后访问，把页面"添加到主屏幕"，即可像 App 一样使用。**

**方式 B：GitHub 集成**

1. 把 `traverse-calc/` 推到 GitHub 仓库
2. 在 [vercel.com](https://vercel.com) 导入该仓库
3. 框架预设选 "Other"，构建命令留空，输出目录 `.`
4. 点 Deploy

**方式 C：Netlify Drop**

访问 [app.netlify.com/drop](https://app.netlify.com/drop)，把整个 `traverse-calc/` 文件夹拖进去，30 秒拿到一个公网 URL。

## 使用说明

**界面布局（移动端优先）**：

- 顶栏（站点名 + 项目操作按钮）
- **sticky mode 切换条**（紧贴顶栏下方）：「闭合」/「附合」两个大按钮，永远可见，附合按钮不会再"点不开"
- 起算数据卡片（已知点点号、坐标、起算方位角）
- **测站角度表**：每行 = 一点（点号 + β 三栏度分秒），可加/删测站
- **边长表**：每行 = 一条线段（起点→终点 label 只读 + 距离输入），行数 = 边数 = 测站数
- 限差 / 角型设置卡片
- **底部 sticky「🚀 计算」按钮**（贴底，iOS safe-area 适配）：按了才重算
- 结果表 / 略图 / 导出

**操作流程**：

1. 选模式（闭合 / 附合）
2. 填起算数据（附合模式还要填终止方位角 + 终止点坐标）
3. 加测站 → 改点号 → 填 β 三栏度分秒 → 填对应边长
4. 改任意输入后，底部「🚀 计算」按钮变橙脉动、显示「已修改 · 点此重算」
5. 点按钮 → 看结果表 + 略图 → 导出

**为什么不实时重算**：

实时重算在大输入表（10+ 测站）上会卡顿，且用户还没输完就刷新会误导。改为按按钮触发后，输入流畅，重算时机明确，结果与输入强对应。

## 核心算法

**闭合导线**：
1. 角度闭合差 `fβ = Σβ - (n-2)·180°`
2. 改正数 `vβ = -fβ/n`，逐角加
3. 推方位角（左角 `α_next = α_prev + β' - 180°`；右角 `-`）
4. 增量 `ΔX = D·cosα`，`ΔY = D·sinα`
5. 坐标闭合差 `fx, fy` → `fs, K`
6. 增量按边长比例分配闭合差
7. 推坐标

**附合导线**：与闭合类似，区别在 `fβ = α_起 ± Σβ ∓ α_终 - n·180°`，`fx, fy` 用已知终点点位反算。

详见 `js/traverse.js`。

### 整数修正模式（largest-remainder）

关闭时按比例精确分配（理论值，浮点）；开启时把 v_β 凑成整秒、vx/vy 凑成 1mm 整数倍，符合测量手工报表习惯。

**算法**：largest-remainder 1D 版
1. `base = trunc(target / unit / n)`，每项先拿 base
2. `remainder = round(target/unit - base*n)`，是 |remainder| 个 |weight|（|β| 或边长）最大的项各 + step·unit
3. `Math.round` 防 `1.000000000045` 浮点漂移（剩 2 项变 1 项的 bug）

**角度**：unit=1 秒，weight=`|β|（度）`
- 例：fβ=-25″，4 角 → base=6，remainder=1，vβ=[+7, +6, +6, +6]（最大 |β| 拿 +1）
- Σ 严格 = -fβ，极差 ≤ 1

**坐标**：unit=0.001 米，weight=`distance`
- 例：fx=+0.009，4 边长 188/118/184/111 → vx=[-0.003, -0.002, -0.002, -0.002]（188m 那边拿 -0.003）
- Σ 严格 = -fx，容许 2mm 累计舍入

实现：`js/traverse.js` 的 `integerDistribute(target, unit, n, weights)`，被 `adjustAnglesAndAzimuths` + `distributeClosure` 透传。

## 验收精度

- 角度限差默认 `±40″·√n`
- K 限差可调（1/2000 / 1/4000 / 1/6000 / 1/8000 / 1/10000 / 1/15000）
- 图根导线一般 1/2000 ~ 1/4000
- 等级导线可达 1/15000 以上

## 已知边界

- 闭合导线至少 3 个测站
- 附合导线至少 2 个测站
- 所有数据本地存储，不上传服务器
